Orders Of Magnitude

  • L1 Cache - 3 Cycles ( 1 ns )

  • L2 Cache - 14 Cycles (5 ns)

  • RAM - 250 Cycles (90 ns)

  • SSD - 270 000 Cycles (90 usec)

  • GigE Lan - 600 000 Cycles (200 usec)

  • Spinning Rust - 6 000 000 (2 msec)

  • Internet - 300 000 000 (100ms)

Don't Block The Event Loop

  • Node Is Single Threaded By Design

    Pay no attention to the man (IO threads) behind the curtain.

  • One Big Loop

    As long as there are events to deal with, it keeps looping, forever.

  • Don't Block This Loop

    Bad things happens. As in nothing else!

No Really, Don't Try To Block

  • But My Fibonacci Sequence Server Is Imporant

    Sorry! If you want to use Node for it, you'll need to be more creative.

  • Node Is Not A Hammer

    Node purposefully makes blocking patterns difficult. Get over it.

  • Ruby, Python, Perl

    These hammers are still great.

  • Use Helper Modules

    There are lots of tools for working with asynchronous JavaScript

Function Scoped Variables

In JavaScript 'var' Variables Are Function Scoped


    //lets print a whole bunch of 5s!
    var myArray =[ 1, 2, 3, 4, 5 ];
    for( var i = 0; i < myArray.length; i++ ) {
        var value = myArray[i]];
        fs.write( fd, myArray[i] )
    }            
    

    //let's print 12345
    var myArray =[ 1, 2, 3, 4, 5 ];
    var write_element = function( i ) {
        var value = myArray[i]];
        fs.write( fd, myArray[i] )
    };
     for( var i = 0; i < myArray.length; i++ ) {
        write_element( i )
     }

Writes Can Be Deceiving

Just because the call back is called, doesn't mean it's made it to disk.


    var on_write = function( err ) {
        if ( ! err ) { console.log('all clear!') };
        //insert power failure here
    };
    fs.write(fd, buffer, offset, length, position, [callback])
    

    
    var on_write = function( err ) {

        var on_fsync = function ( err ) {    
            if ( ! err ) { 
                console.log('according to the OS, I'm all clear!') 
            };
        };  
        fs.fsync( fd, on_fsync ) 
        
        //insert power failure here
    
    };
    fs.write(fd, buffer, offset, length, position, on_write )

Callback Hell

Callback hell can be mitigated.


    //look mom, fewer lines!
    fs.write(fd, buffer, offset, length, position, function( err ) {        
        if ( ! err ) { 
            fs.fsync( fd, function( err ) {        
                if ( ! err ) { 
                    console.log('according to the OS, I'm all clear!') 
                }
            });
        }  
        //insert power failure here
    });  
    

    var on_write = function( err ) {
        if ( ! err ) {
            var on_fsync = function ( err ) {    
                if ( ! err ) { 
                    console.log('according to the OS, I'm all clear!') 
                };
            };  
            fs.fsync( fd, on_fsync ) 
        }
        //insert power failure here
    };
    fs.write(fd, buffer, offset, length, position, on_write );
